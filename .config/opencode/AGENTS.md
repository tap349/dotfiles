## Sisyphus Simplicity Rules

For simple tasks, prefer direct execution over orchestration.

A task is simple when it is:
- a single-file edit
- a typo, rename, comment, or small config tweak
- a direct command the user asked to run
- an explanation-only question

For simple tasks:
- do not create todos
- do not delegate to subagents
- do not consult Oracle or planning agents
- do not run broad searches
- run only the smallest relevant validation, if any
- answer concisely

Use full orchestration only for multi-file changes, ambiguous requirements, risky operations, architecture decisions, frontend/UI work, or user-requested review/QA.
